# SPEC — Мост MAX ↔ Hermes Agent

Бриф для продолжения в Claude Code. Цель: Hermes-агент сам пишет мне в Telegram,
когда в мессенджере MAX приходит новое сообщение, и я могу ответить ему, что с этим
делать — продиктовать ответ или попросить придумать и отправить от моего имени.

## Среда

- Hermes Agent стоит локально на **Windows** (модель локальная). Gateway будет работать,
  только пока машина включена — для уведомлений это ок.
- MAX я использую через браузер (десктопного приложения MAX нет).
- Транспорт к MAX: **PyMax**, pip-пакет `maxapi-python` (версия **2.x**, на момент
  обсуждения 2.1.0). Активный репозиторий — `fresh-milkshake/pymax`
  (`MaxApiTeam/PyMax` — архивное зеркало). `MaxClient` с `device_type="WEB"` даёт токен,
  совместимый с веб-версией.
- ⚠️ Это **неофициальный внутренний API MAX**: может нарушать ToS, риск блокировки
  аккаунта, протокол меняется без предупреждения. Версию пакета **пинить**. По возможности
  тестировать на непервостепенном аккаунте.

## Архитектура (две половины)

**Входящие (агент пишет первым) = cron.** Проактивная доставка в Hermes — это работа
кронового планировщика (тик раз в 60 c). Persistent WS-listener НЕ нужен: 60 секунд
латентности устраивает, PyMax коннектим на каждый тик (сессия в SQLite, переподключение
дешёвое).

**Исходящие (я диктую / прошу придумать и отправить) = тулы плагина.** Обычные тул-вызовы
в моём диалоге с агентом.

### Поток

1. Cron «every 1m» с **pre-check скриптом**: PyMax тянет новое с момента watermark.
   - нет новых → `{"wakeAgent": false}` (тик стоит ноль, модель не будится)
   - есть новые → обновить watermark, вернуть `{"wakeAgent": true, "context": "<сообщения: кто / чат / текст>"}`
2. Разбуженный агент по промпту задачи пишет мне в `telegram:<мой_id>` короткую сводку
   («📨 Маша (чат X): про встречу в пятницу. Ответить?»).
3. Я отвечаю в той же личке: «ответь — буду в семь» или «придумай вежливый ответ и отправь».
   Агент зовёт `max_send`. Вариант «придумай и отправь» — та же механика, просто агент сам
   сочиняет текст.

## Компоненты

### 1. Плагин `~/.hermes/plugins/maxbridge/`
Четыре файла (см. доку «Build a Hermes Plugin»):
- `plugin.yaml` — манифест: name, version, `provides_tools`, опц. `requires_env`.
- `schemas.py` — JSON-схемы тулов (что читает модель).
- `tools.py` — хендлеры. **Правила: сигнатура `def h(args: dict, **kwargs) -> str`,
  ВСЕГДА возвращать JSON-строку (и на ошибке тоже), НИКОГДА не кидать исключение
  (ловить всё → `{"error": ...}`).**
- `__init__.py` — `def register(ctx):` связывает схемы с хендлерами через
  `ctx.register_tool(...)`, при необходимости хуки через `ctx.register_hook(...)`.

Тулы:
- `max_list_chats()` — список чатов.
- `max_history(chat, count)` — история.
- `max_send(chat, text)` — отправка от моего имени.

### 2. Watermark-хранилище
Маленький JSON/SQLite: последнее виденное сообщение на чат. Пишет pre-check скрипт,
чтобы не дёргать меня повторно.

### 3. Cron-задача
Pre-check скрипт (PyMax fetch + watermark + формат `wakeAgent`) + промпт-сводка,
доставка в `telegram:<мой_id>`.

## Открытые вопросы — решить на коде, не угадывать

1. **Связь «сессия крона → сессия моего ответа».** Если контекст «о каком чате речь»
   не переносится автоматически в сессию, где я отвечаю, — добавить хук `pre_llm_call`,
   который подмешивает «MAX-чаты, ждущие ответа» в мой ход. (`pre_llm_call` — единственный
   хук, чьё возвращаемое значение не игнорируется; возвращает `{"context": ...}`.)
   Сначала проверить, шарится ли контекст и так.
2. **Async-хендлер в плагине.** PyMax асинхронный. Подтвердить путь регистрации async-тула
   (во встроенном пути есть `is_async=True`) либо обернуть синхронным мостом. Свериться с
   `developer-guide/tools-runtime`.

## Безопасность (обязательно)
Gateway имеет доступ к терминалу и к моему аккаунту MAX.
- По умолчанию gateway пускает только allowlist / спаренных по DM — оставить так
  (`TELEGRAM_ALLOWED_USERS` или DM-pairing).
- Поставить мою личку home-каналом: `/sethome`.

## Порядок работ
1. **Probe**: установить `maxapi-python` (2.x, пин), `MaxClient(device_type="WEB")`,
   авторизация, один `fetch_history` моим аккаунтом — убедиться, что либа СЕЙЧАС
   говорит с актуальным MAX. Если молчит — дальше нет смысла, разбираться с версией/форком.
2. **Плагин** maxbridge (4 файла + тулы), проверить `hermes` показывает тулы в баннере,
   протестировать `max_send` в тестовый чат.
3. **Cron** с pre-check скриптом и watermark; проверить тихие тики и пробуждение на новом
   сообщении, доставку сводки в телегу.
4. Решить открытый вопрос №1 (контекст ответа) по факту поведения.
